Merge Conflicts


What is a merge conflict? When you work with a team (and even sometimes when you are working alone) you will occasionally create merge conflicts. At first, merge conflicts can be intimidating, but resolving them is actually quite easy.

Resolving a merge conflict

  1. Open that file in your text editor, and look for the merge conflict markers. (<<<<<<<, =======, >>>>>>>)
  2. Both branches' versions of code are present - pick which one you want to keep, and save the changes.
  3. Add and commit the saved changes to resolve the merge conflict.
  4. Push the feature branch up to the remote, and see the resolution in the pull request.
  5. Merge the pull request.
